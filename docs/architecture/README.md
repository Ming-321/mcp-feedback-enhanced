### 🏗️ 架構概覽

MCP Feedback Enhanced 目前採用**多活躍會話 + session_id 綁定頁面**的 Web 架構。每個 Web 頁面都綁定到一個具體會話，不同工作區可以並行保留獨立頁面；同一工作區再次建立新會話時，會復用原頁面並切換到該工作區的最新會話。

#### 核心特性
- **多活躍會話**: 不同工作區可同時保留獨立活躍會話
- **session_id 綁定頁面**: Web 頁面、API 與 WebSocket 都綁定到具體會話，不再依賴全域 current session
- **同工作區頁面復用**: 同一工作區再次建立新會話時，原頁面會跳轉到最新會話
- **顯式歧義處理**: 相容接口在多活躍會話情境下返回明確歧義錯誤，而不是靜默綁錯
- **持久化 Web UI**: 支援多次循環調用，減少重複開頁與上下文丟失
- **智能資源管理**: 自動清理和會話生命週期管理
- **多語言支援**: 繁體中文、簡體中文、英文動態切換

#### 技術棧
- **後端**: Python 3.11+, FastAPI, FastMCP
- **前端**: HTML5, JavaScript ES6+, WebSocket, Web Audio API（v2.4.3）
- **通信**: WebSocket, HTTP REST API
- **存儲**: localStorage（會話歷史、音效文件、設定記憶）
- **部署**: uvicorn, 跨平台支援

### 🆕 最新架構調整（v2.6.x）

#### 🪟 多工作區頁面模型
- **不同工作區並行**: 多個工作區可同時有活躍會話與獨立頁面
- **同工作區替換**: 同一路徑工作區建立新會話時，舊會話會進入 `superseded` 終態，避免舊 MCP 調用掛起
- **工作區顯示名**: 預設使用目錄名；若重名則自動補上上級目錄，必要時再補短 hash

#### 🔌 協議綁定調整
- **頁面路徑**: 新增 `/session/{session_id}` 作為具體會話頁面
- **API 路徑**: 新增 `/api/sessions/{session_id}` 取得指定會話內容
- **WebSocket 路徑**: 新增 `/ws/{session_id}` 綁定具體會話連線
- **相容路徑**: `/`、`/api/current-session`、`/ws` 只在單一活躍會話時有明確語義；多活躍會話時會回傳歧義結果
